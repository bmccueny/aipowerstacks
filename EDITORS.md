# AI PowerStacks: Editor Guidelines

Every blog post on AI PowerStacks should feel like a high-signal, premium briefing. Follow these rules to maintain the "Gumroad-style" brutalist aesthetic.

## 1. Personality & Voice
- **Rowan Cheung:** Speed, tool-heavy, "here's what you missed."
- **Pete Huang:** Humorous, punchy, business-casual, "high signal / low noise."
- **Zain Kahn:** Practical, educational, "how to use this to save 10 hours."
- **Ethan Mollick:** Experimental, visionary, "I tested this and here's the reality."
- **Simon Willison:** Builder-focused, transparent, "here's how it actually works."

## 2. Visual Content (Cover Photos)
Never leave a post without a cover photo. Use these sources:

### A. Automated Tool Screenshots (Preferred for Tool Reviews)
If the post is about a specific tool (e.g., "Cursor vs VS Code"), use the **Suggest Screenshot** button in the editor or manually use Thum.io:
`https://image.thum.io/get/width/1200/crop/600/noanimate/[URL_OF_TOOL]`

### B. Brutalist Stock Photography (Preferred for Essays/Guides)
Source from **Unsplash** using these search terms for a consistent look:
- "Abstract technology"
- "Minimalist 3D"
- "Brutalist architecture"
- "High contrast tech"
- "Vaporwave tech" (for more colorful posts)

### C. Aesthetic Requirements
- **High Contrast:** Images should have strong shadows or clear lines.
- **Minimalism:** Avoid cluttered, busy stock photos with too many people.
- **Primary Colors:** Try to find images with hints of Pink/Rose (matching our theme) or deep Blacks/Whites.

## 3. Post Formatting
- **Title:** Bold and punchy (max 60 chars).
- **Excerpt:** This is the "TL;DR". It must be a 2-sentence summary that provides immediate value.
- **Headings:** Use H2s for major sections and H3s for sub-points.
- **Links:** Always link to the tools mentioned using the `prose-a` style.
