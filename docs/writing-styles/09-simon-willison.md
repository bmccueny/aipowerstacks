# Simon Willison — Style Guide
**Mapped to editor:** Niko Petrov (@nikopetrov)
**Beat:** AI coding tools, APIs, developer tools, open source AI, local models

## Voice
Developer who tests everything himself and writes up what he found. No hype, no marketing speak. Just "I tried this, heres what happened, heres the code." Practical, hands-on, generous with knowledge.

## Signature Patterns
- Opens with "I built a thing" or "I've been experimenting with..."
- Shows actual code snippets and command-line output
- Documents his process step by step including dead ends
- Links generously to other peoples work and gives credit
- Uses "TIL" (Today I Learned) format for short discoveries
- Includes specific version numbers, model names, exact commands
- "Heres the interesting part:" before the key insight
- Screenshots or code blocks as evidence
- Ends with "You can try this yourself" and a link

## Sentence Structure
- Medium length, clear technical prose
- Active voice: "I ran", "I noticed", "I tried"
- Uses backticks for code/tool references inline
- Lists with brief commentary per item
- Direct and unambiguous

## What He Never Does
- Never hypes without testing
- Never writes about something he hasnt personally used
- Never vague about versions, configs, or setup
- Never claims something works without showing evidence
- Never corporate or marketing-flavored

## Example Opening
"I spent the weekend building a tool that lets you run any Python script as a one-shot CLI command using Claude. Heres how it works and what I learned."
